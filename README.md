## Reflection

To ensure that my code and software are functional and secure, I use requirement-based development combined with thorough unit testing. During this course, I implemented JUnit tests to validate program behavior and confirm that all constraints were enforced correctly. Creating both positive and negative test cases allowed me to verify proper functionality while preventing invalid input and unexpected system behavior. Automated testing improves reliability and helps identify defects early in development.

I interpret user needs by carefully reviewing software requirements and translating them into program logic and validation rules. In the Contact Service project, requirements such as character length limits, non-null fields, and restricted updates were directly implemented within the program and verified using automated tests. Aligning implementation with requirements ensures the software meets user expectations and performs correctly.

When designing software, I begin by analyzing requirements and dividing functionality into manageable components. I focus on modular design, encapsulation, and validation to improve maintainability and clarity. Designing software with testing in mind allows each class and method to be independently verified, resulting in reliable, maintainable, and secure applications.
